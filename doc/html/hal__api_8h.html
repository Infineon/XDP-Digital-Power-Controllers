HAL_INLINE uint16_t HAL_GET16 ( volatile uint16_t *  addr)

HAL_SET16() provides the lowest level of memory access: It allows to read a uint16_t value from some address.

This version of HAL_SET16() is meant for the FW image on the actual target silicon - a simple assignment does the job.

Parameters
addrpointer to address, needs to be volatile because there might HW 'behind' this address, changing the address' content.
