Custom RTOS Extensions

A CMSIS-RTOS is held as small as possible and does not foresee any extensions by nature. Though, in some cases it does make sense to extend it, especially for modularization purposes. The highest priority extension is the Structured Exception Handling mechanism for multiple threads which makes it possible to centralize the exception handling in a single module.

This CMSIS-RTOS API extension encapsulates the actual CMSIS-RTOS API so that the latter is not affected by the custom extensions. This in turn leads to the necessity to exchange those functions from the actual CMSIS-RTOS API that need extended functionality by the ones from this API. The essential function for this purpose is osThreadCreate() which must be exchanged by osExtThreadCreate(). In addition where necessary substitute all includes of cmsis_os.h with cmsis_os_ext.h

Note
Nomenclature: All extensions shall follow the coding style of the CMSIS-RTOS API to stay independent from a varying application coding style. The prefix of each extension is os<Moduleprefix>. For instance, the function descriptors of the extension for exception handling look like osExcFunctionName.
