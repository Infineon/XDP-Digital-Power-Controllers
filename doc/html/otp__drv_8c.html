Detailed Description

OTP device access layer.

The driver support interrupt or polling mode to identify if the OTP hardware got programmed successfully. The interrupt mode (requires RTX) ensures that the CPU is released in case we wait for a programming result. The polling mode is useful for environments (e.g. Lauterbach OTP monitor program) in case no RTX with interrupt support is given. The polling mode is automatically activated if the kernel is not running.
