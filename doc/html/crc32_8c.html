Fast CRC32: Half-Byte Algorithm. For certain embedded CPUs the bitwise algorithm might be too slow but there is not enough memory available for the look-up tables of the faster algorithms either (between 1k and 8k). The half-byte/nibble algorithm is a sweet compromise: its look-up table contains only 16 entries for a total of 64 bytes and runs about three times faster than the best bitwise algorithm.
